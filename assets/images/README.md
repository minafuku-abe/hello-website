## 画像の最適化について

画像ファイルサイズが大きい場合は、以下の無料サービスで最適化することをおすすめします：

- **TinyPNG** (https://tinypng.com/) - PNG・JPEG最適化の定番
- **Squoosh** (https://squoosh.app/) - Googleが提供する画像圧縮ツール
- **Optimizilla** (https://imagecompressor.com/) - 複数画像の一括処理対応

## 注意事項

- ファイル形式: JPG または PNG
- ファイル名は必ず上記の通りにしてください（大文字・小文字も区別されます）
- UTF-8エンコーディングで保存してください
- 画像が足りない場合は、似た写真を複数の箇所で使い回しても問題ありません

## 一時的な代替案

もし今すぐ全ての写真が揃わない場合、一部をプレースホルダー画像（無地の色）で代用することもできます。
後から差し替えることも可能です。

## 画像の配置・差し替え方法

### 初回配置

1. 用意した写真を、このフォルダ（`public/assets/images/`）に配置してください
2. ファイル名は上記のリスト通りにしてください（例: `hero-bg.jpg`）
3. ブラウザでページを再読み込みすると、写真が表示されます

**注意:** 写真がない箇所には、プレースホルダー画像が自動的に表示されます。

### 写真の差し替え

1. 差し替えたい写真を、同じファイル名で上書き保存してください
2. ブラウザでページを再読み込み（**Ctrl+F5** または **Cmd+Shift+R**）してください
3. キャッシュをクリアすると確実に新しい画像が表示されます

### プレースホルダーについて

写真を配置する前や、ファイル名が間違っている場合、自動的にプレースホルダー画像が表示されます。

**プレースホルダーの利点：**
- ページのレイアウトが崩れません
- どこに写真を配置すべきか分かりやすくなります
- 段階的に写真を追加できます

**すべての写真を一度に用意する必要はありません。1枚ずつ配置していってください。**

### 配置例

```
public/assets/images/
├── hero-bg.jpg        ← まずこれを配置
├── profile.jpg        ← 次にこれを配置
├── spring.jpg         ← 徐々に追加
├── placeholder.svg    （自動生成済み）
├── placeholder-hero.svg（自動生成済み）
└── placeholder-portrait.svg（自動生成済み）
```

### トラブルシューティング

**Q: 写真を配置したのに表示されない**
- ファイル名が正しいか確認してください（大文字・小文字を区別します）
- ブラウザのキャッシュをクリアしてください（Ctrl+F5 / Cmd+Shift+R）
- ファイル形式がJPGまたはPNGか確認してください

**Q: プレースホルダーが表示され続ける**
- ファイル名のスペルミスがないか確認してください
- ファイルが正しいフォルダに配置されているか確認してください

**Q: 一部の写真だけ差し替えたい**
- 該当する写真ファイルを同じ名前で上書きするだけです
- 他のファイルには影響しません
